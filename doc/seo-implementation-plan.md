# SEO Implementation Plan

## Current State

The website is a static site (HTML/CSS/JS) with shared sidebar, consistent styling via `styles.css`, and a digital store powered by `store/products.json`. It has **zero SEO infrastructure** — no meta tags, no structured data, no sitemap, no language targeting.

The primary SEO goal is to drive organic traffic to the digital store, specifically the "IT for Seniors" ebook targeting Spanish-speaking elderly users.

## Constraints

- Maintain consistent UI across all pages (shared styles.css, sidebar, script.js)
- Keep digital store content parametrizable via products.json
- Static site — no server-side rendering available

## Phase 1: HTML Foundation (all pages)

### 1.1 Add `<head>` meta tags to every page

Each page needs at minimum:

```html
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>[Page-specific title]</title>
  <meta name="description" content="[Page-specific description, 150-160 chars]">
  <meta name="author" content="Marcin Kaminski">
  <link rel="canonical" href="https://mkhome.byst.re/[page].html">
</head>
```

**Files to modify**: `index.html`, `store.html`, `portfolio.html`, `projects.html`, `utilities.html`

### 1.2 Add Open Graph and Twitter Card tags to every page

```html
<meta property="og:title" content="[title]">
<meta property="og:description" content="[description]">
<meta property="og:url" content="https://mkhome.byst.re/[page].html">
<meta property="og:type" content="website">
<meta property="og:image" content="https://mkhome.byst.re/[og-image].png">
<meta name="twitter:card" content="summary_large_image">
```

**Action needed**: Create a generic OG image (1200x630px) for pages without a specific one.

### 1.3 Verify `lang` attribute on all pages

- `index.html`, `portfolio.html`, `projects.html`, `utilities.html`: `lang="en"`
- `store.html`: Currently set dynamically via JS — add `lang="en"` as default in HTML source (crawlers don't execute JS)

**Files to modify**: All HTML files — verify `<html lang="...">` is present in source.

## Phase 2: Store Page SEO (highest impact)

### 2.1 Server-side render product content into store.html

**Problem**: Product cards are rendered entirely by JavaScript. Search engines may not index them.

**Solution**: Generate a static HTML version of the store with product content pre-rendered, while keeping the JSON-driven architecture.

**Approach**: Create a build script (`scripts/build-store.sh` or `scripts/build-store.py`) that:
1. Reads `store/products.json`
2. Generates static HTML product cards as `<noscript>` fallback inside `store.html`
3. JavaScript still handles language switching and dynamic rendering on top

This way:
- Crawlers see full product content in HTML source
- Users with JS get the dynamic multilingual experience
- products.json remains the single source of truth

**Files to create**: `scripts/build-store.py`
**Files to modify**: `store.html` (add noscript product blocks)

### 2.2 Add Schema.org Product structured data

Inject JSON-LD structured data for each product. This can also be generated by the build script from products.json.

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Book",
  "name": "IT for Seniors",
  "author": { "@type": "Person", "name": "Marcin Kaminski" },
  "description": "...",
  "url": "https://www.amazon.com/dp/...",
  "image": "https://mkhome.byst.re/store/assets/it-for-seniors.png",
  "inLanguage": "es",
  "bookFormat": "EBook",
  "offers": {
    "@type": "Offer",
    "availability": "https://schema.org/InStock",
    "url": "https://www.amazon.com/dp/..."
  }
}
</script>
```

**Source**: Generated from `products.json` by the build script.
**Files to modify**: `store.html` (inject JSON-LD blocks)

### 2.3 Fix heading hierarchy on store page

Current: H1 -> H4 (skips H2, H3).

Change to:
- H1: "Digital Store" (keep)
- H2: Product titles (currently rendered as card titles)
- H3: Section labels like "Trust" indicators

**Files to modify**: `store.html` (JS rendering logic + inline styles)

### 2.4 Add hreflang tags for multilingual store

Since the store supports EN/ES/PL, add hreflang annotations so Google knows about language variants. Since all languages are on the same URL (client-side switching), use one of these approaches:

**Option A (recommended)**: Create separate store URLs per language:
- `store.html` (default, English)
- `store.html?lang=es` or `store-es.html`

Then add hreflang link tags:
```html
<link rel="alternate" hreflang="en" href="https://mkhome.byst.re/store.html">
<link rel="alternate" hreflang="es" href="https://mkhome.byst.re/store.html?lang=es">
<link rel="alternate" hreflang="pl" href="https://mkhome.byst.re/store.html?lang=pl">
```

**Option B (simpler)**: Keep single URL, set default lang to Spanish (since that's the target market), add meta tags in Spanish.

**Recommendation**: Option A gives better SEO targeting but requires the build script to generate per-language static content. Option B is faster to implement. Start with B, upgrade to A later.

## Build Script Specification (Phase 2.1)

A Python script `scripts/build-store.py` that:

1. **Input**: Reads `store/products.json`
2. **Processing**:
   - For each product, generates static HTML card markup (matching the JS-rendered output)
   - Generates JSON-LD Schema.org blocks for each product
   - Generates `sitemap.xml` from all `.html` files in root
3. **Output**:
   - Injects `<noscript>` product cards into `store.html` (between marker comments)
   - Injects `<script type="application/ld+json">` blocks into `store.html`
   - Writes `sitemap.xml`
4. **Constraints**:
   - Does NOT modify the JS rendering logic
   - Uses marker comments in store.html to know where to inject (e.g., `<!-- SEO:PRODUCTS -->...<!-- /SEO:PRODUCTS -->`)
   - Idempotent — safe to run repeatedly
   - Default language for static content: Spanish (primary target market)

This script should be run after any products.json update (can be added as a Mireq cron job).
